# Google IT Automation: Introduction to Git and GitHub

This repository contains my practical notes, scripts, and documentation from the **Google Introduction to Git and GitHub** course. It serves as a comprehensive knowledge base for version control, system maintenance, and collaborative software development.

## 🎓 Course Syllabus & Coverage

The content is organized according to the core modules of the Google curriculum:

### 1. Before Version Control (System Maintenance)
* **Diffing Files:** Using `diff` and `diff -u` to find changes.
* **Patching:** Applying fixes to original files using the `patch` command.
* **Practical Debugging:** Identifying logic and syntax errors in Python automation scripts.

### 2. Using Git Locally
* **Git Lifecycle:** Understanding the *Working Tree*, *Staging Area (Index)*, and *Git Directory*.
* **Basic Commands:** `git init`, `git add`, `git status`, `git commit`.
* **Undoing Changes:** Using `git checkout`, `git reset`, and `git commit --amend`.
* **Branching & Merging:** Managing independent lines of development and resolving merge conflicts.

### 3. Working with GitHub
* **Remote Repositories:** Working with `git clone`, `git push`, and `git pull`.
* **Collaboration:** Managing Pull Requests (PRs), code reviews, and remote branches.
* **GitHub Workflow:** Forking repositories and contributing to open-source projects.

### 4. Collaboration & Real-World Scenarios
* **Merge Conflicts:** Identifying and resolving conflicting changes manually.
* **Rollbacks:** Reverting faulty commits in a production environment.
* **Rebasing vs. Merging:** Keeping a clean project history.
